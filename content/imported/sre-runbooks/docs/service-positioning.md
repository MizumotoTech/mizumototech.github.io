# Service Positioning

## One-line Positioning

Mizumoto Tech helps teams build cloud infrastructure foundations on GCP with Terraform
and CI/CD, with clear handover documentation.

## Service Scope (v0.2)

### Primary Service Packs

1. Terraform Foundation Pack
2. CI/CD Bootstrap Pack

### Entry Offer

1. Architecture Review Pack

### Add-ons

1. Runbook and Handover Add-on
2. Cloud Cost Hygiene Pack

## Phase-1 Delivery Boundary

- Delivery promise is GCP-first in Phase 1.
- Multi-cloud patterns may be added later as examples, not as initial
  service commitments.

## Offer Statements

- Terraform foundation setup for `dev` / `stg` / `prd`, so teams can
  provision environments consistently without ad-hoc manual setup.
- CI/CD bootstrap with safe deploy flow, so deployments are repeatable,
  reviewable, and easier to roll back.
- Architecture review with prioritized action plan, so teams can decide what
  to fix first instead of collecting generic best practices.
- Runbook and handover assets as add-ons, so day-2 ownership does not depend
  on the original builder.
- Cost hygiene baseline as an add-on, so obvious spend issues can be found
  before deeper FinOps work starts.

## Not Primary Offer (for now)

- Full observability overhaul.
- SLO and error budget program design.
- Incident response retainer as the core product.
- Multi-cloud implementation from day one.

## Private and Public Policy

### Default Non-Public Assets

- Working notes and internal drafts.
- Client-derived artifacts and review comments.
- Environment-specific commands.
- Billing, IAM, org inventory, and internal process details.

### Public-Eligible Assets (after sanitization)

- General service definitions.
- Reusable templates.
- Generalized case studies.
- Public-safe runbook examples.
- Reference architecture examples without client identifiers.

### Sanitization Rules

- Remove client names, project IDs, cost numbers, and private endpoints.
- Replace organization-specific values with placeholders.
- Keep only reusable patterns and neutral examples.
- When in doubt, generalize the pattern first before publishing.

## Why This Positioning

- It matches existing experience and current strengths.
- It creates tangible outputs clients can verify quickly.
- It avoids overpromising in weak-fit domains.
