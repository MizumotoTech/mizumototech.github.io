# Terraform Foundation Pack

## 摘要

该服务包为中小型 GCP 团队提供可复用的 Terraform 基线，覆盖 `dev`、
`stg`、`prd`，使基础设施变更更易评审、更安全执行、也更易交接。

## 解决的问题

- 环境搭建依赖一次性手工操作。
- Terraform state 与模块边界不清晰。
- 新接手成员无法在缺少口头说明时安全操作。

## 理想客户画像

- 使用 GCP 的中小型产品团队。
- 需要 `dev` / `stg` / `prd` 环境治理。
- 希望先完成标准化再扩展复杂基础设施的团队。

## 服务范围

- 多环境 Terraform 的仓库布局与基线工作流。
- Remote state 设计与环境状态隔离模式。
- 环境初始化所需的基础 IAM 与 service account 模式。
- 可扩展的共享模块与变量结构。
- 环境 bootstrap 指南与评审流程。
- 面向 day-2 的交接文档。

## 实施边界

- 本服务包聚焦于基础设计、基线结构与交接。
- 除非明确约定，否则不默认包含完整生产落地。
- 超出约定基线范围的定制模块开发作为后续工作处理。
- 既有未纳管资源的迁移不包含在初始服务包内。

## 非目标

- 全组织级 landing zone 重构。
- 多云生产级并行实施。
- 7x24 运维值守与事故响应。
- 深度可观测性与 SLO 体系落地。

## 客户需提供输入

- 目标 GCP 项目与环境规划。
- IAM 边界与安全约束。
- 现有仓库和 CI/CD 约束。
- 必需云服务与区域约束。

## 客户责任

- 按约定时间提供所需权限与环境信息。
- 在评审窗口内确认项目、区域、命名与 IAM 决策。
- 指定至少一名评审人负责架构与变更确认。
- 在约定时间内完成交接包评审与签收。

## 交付物

### 文档交付

1. Terraform 基线定义文档。
2. 命名、state 与变量约定指南。
3. `init` / `plan` / `apply` 执行手册。
4. 风险控制、回滚说明与变更门禁。
5. 面向 day-2 的交接指南。

### 基线技术交付

1. Terraform 基线仓库蓝图。
2. 可扩展的基础模块结构。
3. `dev` / `stg` / `prd` 环境布局骨架。
4. 示例命令流与配置占位样例。

## 交接内容

- 环境接入检查清单。
- 标准操作说明（`plan`、`apply`、评审流程）。
- 变更审批与回滚决策点。
- 已知限制与后续 backlog。

## 验收标准

1. 范围与环境边界文档明确且无歧义。
2. `dev` / `stg` / `prd` 的 state 与模块结构定义完成。
3. 执行流程含可复制命令示例。
4. 文档包含 `fmt`、`validate`、`plan` 校验步骤。
5. 回滚条件与停止条件明确写出。
6. 指定工程师可仅依据交接文档完成非生产验证流程，且无需依赖未文档化
   的口头步骤。

## 前提假设

- 客户可在约定时间内提供所需云权限。
- 团队可在项目窗口内完成架构决策评审。
- 初始服务包不包含既有生产负载迁移。

## 风险与约束

- IAM 审批延迟会阻塞前期验证。
- 既有命名不一致会增加规范化工作量。
- 隐性依赖可能导致后续补充阶段。

## 不包含项

- 生产迁移执行。
- 事故响应执行。
- 超出成本卫生基线的优化实施。

## 定价模型占位

- 推荐模式：固定范围包 + 里程碑付款。
- 可选扩展：限时 follow-up 实施区块。
- 最终报价：在 discovery 与范围确认后确定。
- 示例里程碑：discovery 与环境映射。
- 示例里程碑：基础设计与蓝图交付。
- 示例里程碑：基线交接包签收。

## 成功信号

- 环境边界清晰，可避免临时手工搭建。
- Terraform 变更可按固定评审与执行流程推进。
- day-2 接手方可快速定位 state、变量与模块结构。
- 后续基础设施工作可在现有基线扩展而非重做。

## 可选附加项

- Runbook 与交接深化包。
- CI/CD Bootstrap Pack 集成。
- 成本卫生评估包。
