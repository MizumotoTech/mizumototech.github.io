# CI/CD Bootstrap Pack

## 摘要

该服务包为中小型 GCP 团队提供可复用的 CI/CD 基线，从 build 到
deploy 的发布更易评审、更安全晋级、也更易回滚。

## 解决的问题

- 发布依赖手工或不一致的流程。
- 环境晋级规则不清晰。
- 流水线失败后处理方式临时化，缺少稳定回滚指引。
- 新接手成员若缺少文档化知识，难以安全操作发布流程。

## 理想客户画像

- 使用 GCP 的中小型产品团队。
- 已有开发流程，但缺少稳定的发布工作流。
- 希望先建立发布安全边界，再提升交付速度的团队。

## 服务范围

- 覆盖 build、test、镜像发布、deploy 的 CI/CD 基线设计。
- 非生产与生产环境的发布分层模式。
- 生产变更审批门禁模式。
- 流水线密钥与凭证处理模式。
- 面向安全运行的失败处理模式与回滚检查点。
- 面向 day-2 的交接文档。

## 实施边界

- 本服务包聚焦于启动基线设计、工作流骨架和交接。
- 除非明确约定，否则不默认包含全量 SDLC 改造。
- 遗留流水线迁移默认属于后续工作，除非明确纳入范围。
- 复杂测试策略重构不在初始服务包范围内。

## 非目标

- 全量平台工程体系改造。
- 全链路流水线可观测性重构。
- 组织级发布治理体系重构。
- 7x24 发布运维支持。

## 客户需提供输入

- 源代码仓库与 CI/CD 平台约束。
- 目标运行时与部署目标约束。
- 环境与分支或标签策略决策。
- 密钥管理与安全合规要求。
- 变更审批与发布策略要求。

## 客户责任

- 按约定时间提供仓库、执行器与环境访问权限。
- 在评审窗口内确认发布策略与审批规则。
- 指定至少一名评审人负责流水线变更决策。
- 在约定时间内完成交接包评审与签收。

## 交付物

### 文档交付

1. CI/CD 基线设计文档。
2. 分支、标签、手动发布路径触发矩阵。
3. 审批与密钥处理指南。
4. 回滚与失败响应指南。
5. 面向 day-2 的交接指南。

### 基线技术交付

1. build、test、deploy 分阶段流水线蓝图。
2. 非生产与生产阶段骨架。
3. 生产审批门禁基线配置。
4. 示例任务模板与配置占位样例。

## 交接内容

- 流水线接入检查清单。
- 正常发布与失败处理的标准操作说明。
- 审批与回滚决策点。
- 已知限制与后续 backlog。

## 验收标准

1. 触发、环境与晋级规则文档清晰明确。
2. 非生产发布流程包含可重复的 build、验证与部署步骤。
3. 生产发布路径包含明确的人工审批检查点。
4. 密钥处理模式文档化，仓库内无明文密钥。
5. 失败路径包含明确停止条件，可阻断失控的生产发布。
6. 指定工程师可仅依据交接文档完成非生产验证与生产门禁模拟，
   且无需依赖未文档化步骤。

## 前提假设

- 客户可在约定时间内提供所需 CI/CD 与云访问权限。
- 团队可在项目窗口内完成发布策略评审。
- 在签收前，既有生产发布流程保持不变。

## 风险与约束

- 权限缺失会导致基线验证延迟。
- 分支策略不清会阻塞触发和晋级规则确认。
- 遗留发布依赖可能需要后续范围补充。

## 不包含项

- 所有历史流水线的全量迁移。
- 端到端测试框架重构。
- 发布值守执行服务。

## 定价模型占位

- 推荐模式：固定范围包 + 里程碑付款。
- 可选扩展：限时实施与稳定化支持区块。
- 最终报价：在 discovery 与范围确认后确定。
示例里程碑：

1. Discovery 与发布流程映射。
2. 基线设计与流水线蓝图交付。
3. 交接包签收。

## 成功信号

- 发布流程在约定团队和环境范围内保持一致。
- 生产变更可通过明确审批门禁。
- 发布失败具备明确回滚路径。
- day-2 接手方可在无口头依赖下操作流水线。
